# 制約仕様（Constraints）

## 目的

プロジェクトの制約（安全性、品質、運用上の制限）を定義する。本ファイルは制約の仕様正本であり、実装は本ファイルに追従する。

## 制約の適用範囲

- すべての処理は、実行前に制約を評価する。
- 判断不能な場合は安全側に倒す（フェイルクローズ）。
- 制約判定は一箇所に集約し、回避できない構造にする。

## 制約一覧

> しきい値はプロジェクト固有に設定する。初期値は ADR で根拠を明記する。

### C-001 MEXT取得URL制約

| パラメータ | 値 | 備考 |
| ---------- | --- | ---- |
| 許可ドメイン | `mext.go.jp` / `*.mext.go.jp` | SSRF対策（外部任意ドメイン遮断） |
| 許可スキーム | `http`, `https` | `file://`, `javascript:` などを拒否 |
| URL種別 | `.pdf` 拡張子のみ | FR-004 の対象データを限定 |

- 判定：上記のいずれかを満たさない URL は拒否し、処理を継続しない（フェイルクローズ）。
- 理由コード：`C001_INVALID_SOURCE_URL`

### C-002 内部アドレス遮断制約

| パラメータ | 値 | 備考 |
| ---------- | --- | ---- |
| 拒否ホスト接頭辞 | `127.`, `10.`, `192.168.`, `169.254.`, `localhost`, `0.0.0.0`, `::1` | ローカル・リンクローカル・管理ネットワーク遮断 |
| 拒否レンジ | `172.16.0.0/12` | `172.16` 〜 `172.31` を拒否 |
| 実在確認タイムアウト | 5秒（HEAD/GET） | 外部接続の長時間ブロックを回避 |

- 判定：内部アドレスまたは拒否レンジに該当する URL はアクセス禁止とし、検証結果を不合格とする。
- 理由コード：`C002_BLOCKED_INTERNAL_TARGET`

## 状態遷移（必要な場合）

```text
RUNNING ──(品質低下)──→ DEGRADED ──(制約違反/異常連鎖)──→ HALTED
   ↑                      │                               │
   └────(手動復帰)────────┘                               │
   └──────────────(手動復帰)───────────────────────────────┘
```

| 状態     | 説明     | 新規操作             |
| -------- | -------- | -------------------- |
| RUNNING  | 通常稼働 | 許可（制約評価あり） |
| DEGRADED | 品質低下 | 制限                 |
| HALTED   | 安全停止 | 禁止                 |

## 実装要件

- 制約判定は一箇所に集約し、ドメインロジックが回避できない構造にする
- 判定結果には拒否理由コードを付与する（監査ログ用）
- 制約は境界値テストを必須とする（しきい値ちょうど、直上、直下）
- 状態遷移のテストを必須とする（該当する場合）

## 見直し

- しきい値は検証結果に基づき更新する
- 更新時は plan と ADR の整合を取る
